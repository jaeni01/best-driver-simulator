# 0001. 차량 물리 모델로 자전거 모델(Bicycle Model) 채택

## 상태
Accepted (2026-04)

## 맥락
운전면허 2종 보통 도로주행·주차 시뮬레이터에서 차량의 움직임을 계산할 물리 모델이 필요했다.
초기의 단순 점(point) 이동 모델은 차량이 미끄러지듯 비현실적으로 움직여, 조향각에 따른
회전 반경이 표현되지 않았다. 면허시험 도메인의 특성상 고속 주행이 아니라 **저속 정밀 조작**
(T자·평행 주차, 코스 내 회전)이 핵심이므로, 화려한 물리보다 예측 가능하고 사실적인
저속 거동이 더 중요하다고 판단했다.

## 결정
휠베이스와 조향각으로 회전 반경을 계산해 차량의 진행 방향(heading)을 갱신하는
**자전거 모델(Bicycle Model)**을 채택했다. `physics.h`의 물리 상수와
`SimEngine::tick()`(sim_engine.cpp)에서 기어별(P/R/N/D) 거동 분기, 가·감속 적분,
시동 OFF 시 속도 자연 감쇠와 결합해 구현했다.

## 검토한 대안
- **점(point) 이동 모델**: 구현이 가장 단순하지만 조향각·휠베이스가 반영되지 않아
  차량이 미끄러지듯 움직인다. 주차·코스 주행의 조작감을 표현할 수 없어 기각.
- **4WD(4륜) 모델**: 프로젝트 중반에 전환을 검토했다. 바퀴별 동역학까지 표현할 수 있으나
  저속 시험 상황에서는 복잡도만 높고 체감 이점이 적어, 비용 대비 효과가 낮다고 판단해 기각.

## 결과
- 저속 면허시험 상황에 맞는 사실적 주행감을 단일 모델로 확보했고, 휠베이스·조향각·속도라는
  적은 수의 파라미터로 거동을 예측·튜닝할 수 있었다.
- 트레이드오프: 타이어 슬립·하중 이동 같은 고속 동역학은 표현하지 못한다. 다만 본 프로젝트의
  주행 속도 범위에서는 요구되지 않는 특성이라 수용했다.
- 4WD 전환 검토를 거쳐 유지를 선택한 과정에서, 모델 복잡도와 프로젝트 목적 사이의 균형을
  기준으로 기술 의사결정을 내리는 경험을 얻었다.
